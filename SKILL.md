## 搜索方法

**使用 `web_fetch` 工具**访问各平台搜索页。

每个平台并行调用（同时发起）：

```
web_fetch("https://search.bilibili.com/all?keyword=关键词")
web_fetch("https://www.zhihu.com/search?type=content&q=关键词")
web_fetch("https://s.weibo.com/weibo?q=关键词")
web_fetch("https://github.com/search?q=关键词&type=repositories")
web_fetch("https://www.google.com/search?q=site:xiaohongshu.com+关键词")
web_fetch("https://www.google.com/search?q=关键词")  // 全网
```

**每次搜索后自动生成一行 TL;DR**：「💬 一句话：<综合各平台的一句话结论>」

**中文搜索用中文词，英文/技术搜索用英文词。**

**智能扩展**：搜索前自动补全同义词和双语变体。从搜索结果标题中提取高频词作为子话题关键词。

**搜索引擎容错**：优先用 Google，如果 Google 不可用，自动切换到 Bing (`bing.com/search?q=`) 或 DuckDuckGo (`duckduckgo.com/?q=`)。

## 结果提取方法

每个平台返回后，从 HTML 文本中提取：
- **标题**：取 `<title>` 或搜索结果标题行
- **摘要**：取搜索结果的描述文字
- **热度**：从播放量/点赞/评论数估算 → 🔥🔥🔥 (万级以上) 🔥🔥 (千级) 🔥 (百级)
- **立场**：从标题用词判断 → 🟢正面(推荐/好用) 🟡中性(教程/介绍) 🔴批判(吐槽/避雷) 🔵技术 🔣商业
- **时间**：取发布日期

**如果某平台返回空或失败**，标注「⚠️ 未获取到结果」，继续搜其他平台，不中断。

**中文编码处理**：部分中文网站（知乎、B站）的搜索结果可能因编码问题返回乱码。遇到乱码时：
1. 尝试从乱码中提取仍可读的中文字符片段
2. 标注「⚠️ 编码限制，部分内容无法完整提取」
3. 优先使用能正常返回结果的平台

## 深度研究

**阶段一 广撒网**：全网+2平台并行搜索 → 从结果标题提取5-10个高频词作为子话题

**阶段二 深挖**：每个子话题选最优平台单独搜索 → 可信度标注：
- ✅✅✅ 3个以上来源一致
- ✅✅⚠️ 2个来源一致
- ✅⚠️⚠️ 仅1个来源
- 🔴 来源间说法矛盾

**阶段三 分析**：构建时间线（按发布日期排）、标注立场、合并重复内容（标题/URL高度相似的合并为一条）

**阶段四 合成**：输出结构化报告 → `remember("research-<主题>")` 保存

## 搜索历史与监控

- 每次搜索后 `remember(name="search-<词>", type="reference")`

- 深度研究 `remember(name="research-<主题>", type="reference")`
- 每日简报 `remember(name="briefing-<日期>", type="reference")`

依赖 super-memory 技能的 `remember` 工具。

### 智能去重
新搜索前先查历史：`memory(operation="search", query="<关键词>")`。
- 如果 1 小时内搜过同一关键词 → 返回缓存结果，标注「🕐 1小时前搜过」
- 如果搜过相关话题 → 附上「📎 你之前研究过：<相关记忆>」
